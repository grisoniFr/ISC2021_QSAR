Validation procedures

For all regression methods, cross validation can be performed with venetian blinds or contiguous blocks (cross-validation type). As an example, the split based on venetian blinds with 3 cross-validation groups would be for the first group [t,0,0,t,0,0,....,t,0,0], while the second one will be [0,t,0,0,t,0,....,0,t,0], and so on, where t are the samples included in the cross-validation groups. On the other hand, the split of the first group with contiguous blocks will be [t,t,t,t,0,0,....,0,0,0] and so on. If cross-validation is performed, the number of cross-validation groups must be defined. Moreover, bootstrap with resampling or validation based on random sampling (montecarlo) of 20% of samples can be calculated. If bootstrap or montecarlo validation are selected, the number of iterations must be defined.

Ordinary Least Squares (OLS)

In order to calculate regression models based on Ordinary Least Squares (OLS), select "calculate->Ordinary Least Squares ". The form to select the OLS options will appear. Here you can select the type of validation.

[-> top]


Partial Least Squares (PLS)

In order to evaluate the optimal number of latent variables to be retained for Partial Least Squares (PLS), select "calculate->Partial Least Squares ->optimal components for PLS". The form for settings PLS options will appear. Here you can select data scaling and the type of cross validation. The cross validation procedure for selecting the optimal number of latent variables for PLS will produce a plot of the number of components retained in the model versus R2cv and RMSECV. See the example provided in this help to better understand. Finally, in order to calculate the PLS model, select "calculate -> Partial Least Squares ->fit PLS". The corresponding setting form will appear. Here you can select the number of components to be used, data scaling and type of validation.

Principal Component Regression (PCR)

In order to evaluate the optimal number of principal components to be retained for Principal Component Regression (PCR), select "calculate->Principal Component Regression -> optimal components for PCR ". The form for settings PCR options will appear. Here you can select data scaling and the type of cross validation. The cross validation procedure for selecting the optimal components for PCR will produce a plot of the number of components retained in the model versus R2cv and RMSECV. See the example provided in this help to better understand. Finally, in order to calculate the PCR model, select "calculate -> Principal Component Regression ->fit PCR". The corresponding setting form will appear. Here you can select the number of components to be used, data scaling and type of validation.

Ridge regression

In order to evaluate the optimal Ridge parameter (K), select "calculate-> Ridge regression->optimal k for Ridge regression ". The form for settings Ridge options will appear. Here you can select the type of validation. The cross validation procedure for selecting the optimal k will produce a plot of the k values versus R2cv and RMSECV. Finally, in order to calculate the Ridge model, select "calculate->Ridge regression ->fit Ridge regression". The corresponding setting form will appear. Here you can select the K Ridge parameter and type of validation.

Local regression: K Nearest Neighbours (KNN) and Binned Nearest Neighbours (BNN)

In order to evaluate the optimal number of neighbours or the optimal value of alpha for K Nearest Neighbours (KNN) and Binned Nearest Neighbours (BNN), select "calculate-> local regression->optimal parameter for local regression". The form for options will appear. Here you can select the type of local modelling (KNN or BNN), data scaling, type of distance, type of validation. The cross validation procedure for selecting the optimal parameter will produce a plot of the parameter values (K for KNN or alpha for BNN) versus R2cv and RMSECV. Finally, in order to calculate regression models based on local regression, select "calculate->local regression->fit local regression". The corresponding form will appear. Here you can select the method parameter, data scaling, type of distance and type of validation.

Variable selection

All regression methods can be coupled with variable selection approaches: Forward Selection (FS), Genetic Algorithms (GA) and Reshaped Sequential Replacement (RSR). In order to couple variable selection to a specific method, select the desired regression approach from the calculate menu and then select "variable selection" in the option window. The window to set the variable selection options will appear. Here the type of variable selection (FS, GA or RSR) can be chosen, as well as options for each type of selection: number of runs (itarations), maximum number of variables to be selected, percentage of samples to be used as test, number of windows (intervals) for spectral datasets when dealing with GA, minimum and maximum number of variables to be selected, number of seeds, and activation of tabu list for RSR, and maximum number of variables to be selected and number of windows (interval) for FS..


During the Genetic Algorithms (GA) evolution, a figure will appear. This figure will include two plots; the first plot shows a bar plot representing the frequency of selection of each variable, the second plot represents, for each GA run, the R2cv and R2 on test samples (if available) achieved by the best model in the GA population. At the end of the evolution, a new window will appear. In this window, a bar plot representing the frequency of selection of each variable in the best model of each GA run is shown. The user can thus select a frequency threshold; variables associated to frequencies higher than this threshold can then be used for a subsequent forward selection or all subset selection. After the subsequent selection on the reduced set of variables, a new window will appear. Here a list of models is displayed. For each model, the size, R2, R2cv, RMSE, RMSECV and the labels of selected variables are reported. The user can thus select which model to use with the corresponding combo box.


When using Forward Selection (FS) or Reshaped Sequential Replacement (RSR), after the selection, a new window will appear. Here a list of models is displayed. For each model, the size, R2, R2cv, RMSE, RMSECV and the labels of selected variables are reported. As for GA, the user can then select which model to use with the combo box.
